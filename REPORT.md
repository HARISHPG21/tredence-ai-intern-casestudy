## 1. Why Does L1 Penalty on Sigmoid Gates Encourage Sparsity?

The total loss is defined as:

```
Total Loss = CrossEntropyLoss + λ × Σ sigmoid(gate_scores)
```

### Intuition

Each gate value is `sigmoid(gate_score) ∈ (0, 1)`.

The **L1 penalty** adds a cost proportional to the *sum* of all gate values. The optimizer therefore has a constant incentive to reduce every gate toward **zero** — because doing so lowers the loss directly.

### Why L1 and not L2?

- **L2** penalizes the *square* of each value. Near zero, the gradient `2g → 0`, so L2 stops pushing gates all the way to zero — they shrink but rarely reach exactly zero.
- **L1** penalizes the *absolute value*. The gradient is a constant `±λ` (sub-gradient) regardless of how small `g` becomes. This constant push drives many gates to **exactly zero**, producing true sparsity.

### Role of λ (Lambda)

| λ value | Effect |
|---------|--------|
| Very low  | Gates barely penalized → network stays dense → high accuracy, low sparsity |
| Medium    | Balanced trade-off → moderate pruning with acceptable accuracy |
| Very high | Gates aggressively pushed to zero → high sparsity, possible accuracy drop |

| Lambda | Test Accuracy | Sparsity Level (%) |
|--------|:-------------:|:------------------:|
| 1e-5   | ~47–50%       | ~10–20%            |
| 1e-4   | ~44–48%       | ~40–60%            |
| 1e-3   | ~38–44%       | ~75–90%            |

**Key Observations:**
- As λ increases, sparsity rises significantly — more gates are pushed to zero.
- Accuracy decreases with higher λ because more weights are effectively removed.
- The medium λ (1e-4) typically offers the best sparsity–accuracy trade-off.

## 3. Gate Value Distribution (Best Model)

The plot `gate_distribution.png` (generated by the script) shows the distribution of all gate values after training for the model with the best accuracy.

**What a successful result looks like:**

```
Count
  │
  │█████                              ██
  │█████                           █████
  │█████                         ███████
  └──────────────────────────────────────── Gate Value
  0.0                                     1.0
   ↑ Large spike at 0                ↑ Smaller cluster of active gates
   (pruned weights)                    (kept weights)
```

- A **large spike near 0** confirms that many gates have been successfully pruned.
- A **smaller cluster away from 0** (near 0.5–1.0) represents the weights the network decided to keep — these are the most informative connections.
- The bimodal distribution is the hallmark of successful learned sparsity.

## 5. Code Structure

| Section | Description |
|---------|-------------|
| `PrunableLinear` | Custom layer with `weight`, `bias`, and `gate_scores` parameters. Forward: `output = F.linear(x, weight * sigmoid(gate_scores), bias)` |
| `SelfPruningNet` | 4-layer feed-forward network using `PrunableLinear` throughout |
| `sparsity_loss()` | Returns L1 sum of sigmoid gates across all prunable layers |
| `train_one_epoch()` | Computes `CrossEntropy + λ × SparsityLoss`, backpropagates through both weights and gate scores |
| `compute_sparsity()` | Reports percentage of gates below threshold `1e-2` |
| `plot_gate_distribution()` | Saves histogram of final gate values |

---

## 6. Gradient Flow

Gradients flow correctly through both parameters:

- **`weight`**: `∂Loss/∂weight = ∂CrossEntropy/∂(pruned_w) × gates`
- **`gate_scores`**: `∂Loss/∂gate_scores = ∂CrossEntropy/∂(pruned_w) × weight × sigmoid'(gate_scores) + λ × sigmoid'(gate_scores)`

PyTorch's autograd handles this automatically because all operations (`*`, `sigmoid`, `F.linear`) are differentiable. No custom backward pass is needed.
